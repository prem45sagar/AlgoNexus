# Project Report: AlgoNexus
**A Premium Technical Mastery Platform**

## 1. THE PROBLEM STATEMENT
### How did we come up with this?
During technical interview preparation, developers often face "Context Switching Fatigue." Learning resources are fragmented:
*   **Problems** are scattered across platforms like LeetCode and Codeforces.
*   **Notes** live in Notion, Obsidian, or physical notebooks.
*   **Learning Paths** are compiled lazily across disparate YouTube videos and blog posts.
*   **Progress Tracking** is often nothing more than a mental checklist.

**AlgoNexus** was designed directly to mitigate this. We set out to build a **Unified Learning Workspace** that prioritizes **Privacy**. We crafted a dedicated "Second Brain" where your data never leaves your device, keeping your notes and progress 100% secure.

---

## 2. THE TECH STACK
### Why this Local-First architecture?
*   **IndexedDB (Browser Database)**: Selected as our primary storage layer. This local-first approach ensures that user data is stored directly in the browser, providing offline capability and total privacy without the need for a centralized database.
*   **React 19 & Vite**: We chose the lightning-fast Vite bundler combined with React to provide a smooth, immersive Single Page Application (SPA) experience that operates identically to a high-end desktop tool.
*   **Express.js (Scraper Middleware)**: Forms our specialized API layer. It executes server-side logic to natively translate algorithm problem descriptions imported from valid URLs into rich, markdown-ready format, bypassing CORS restrictions.
*   **CSS Architecture (TailwindCSS 4)**: Adopted for utility-first implementations, dynamically structuring our "Brutalist-Modern" aesthetic encompassing dark modes and glassmorphic interfaces.

---

## 3. HIGH-LEVEL DIAGRAM
### How we are solving the problem
The architecture utilizes a **Local-First** pattern:

```text
[ USER INTERFACE (React SPA) ] <----> [ BROWSER STORAGE (IndexedDB) ]
           |                                (100% Private Data)
           |
           |---- (Axios) ----> [ SCRAPER API (Express) ] ----> [ LEETCODE/EXTERNAL ]
```
**The Solution**: By keeping the data in the user's browser, we eliminate the risk of server-side data breaches. The server's only job is to serve the code and act as a bridge for importing new problems.

## 4. DEMO OF THE PROJECT
### Core Modules
*   **The Dashboard**: Analyzes and aggregates state across your local datasets rendering a high-level view of your mastery progress.
*   **Problems Module**: A centralized tracker where you can dynamically auto-import DSA problems via the Scraper API, categorize them, and track progress.
*   **Notes Module**: Your technical documentation engine featuring a robust rich markdown-processor and nested folder organization.
*   **Roadmaps Module**: Specialized tracker rendering iterative milestone logic to deconstruct complex subjects.

## 6. SECURITY & PRIVACY
### Built for the User
1.  **Zero-Server Data**: Unlike traditional MERN apps, AlgoNexus doesn't store your secrets on a central server.
2.  **XSS Protection**: All user-generated and imported markdown is sanitized on the fly using `rehype-sanitize`.
3.  **Sync Security**: Data transfer between devices is handled exclusively through encrypted Sync Codes generated by the user.
